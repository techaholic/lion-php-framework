A listbox component represents a control to display a list of items, showing more than one item at the same time.

Listbox tag is listbox

i.e. 

  1.    <comp:listbox name="categories"/>
    2.

A listbox is a subclass of __ItemListComponent, which is compound by a list of __ItemComponent instances.
A __ItemComponent is a component containing a pair text, value.

We can setup the listbox items by ussing the "item" tag:

i.e. 

  1.    <comp:listbox name="categories">
    2. 

  2.      <comp:item value="1" text="Agriculture, forestry and fishing"/>
    3. 

  3.      <comp:item value="2" text="Arts, sports and recreation"/>
    4. 

  4.      <comp:item value="3" text="Catering and accommodation"/>
    5. 

  5.      <comp:item value="4" text="Construction"/>
    6. 

  6.      ...
    7. 

  7.    </comp:listbox>
    8.

A listbox shows more than one item at the same time. By default, it show 3 items, but we can customize it by setting the rows property.

i.e. 

  1.    <!-- showing 10 categories at the same time -->
    2. 

  2.    <comp:listbox name="categories" rows="10">
    3. 

  3.      <comp:item value="1" text="Agriculture, forestry and fishing"/>
    4. 

  4.      <comp:item value="2" text="Arts, sports and recreation"/>
    5. 

  5.      <comp:item value="3" text="Catering and accommodation"/>
    6. 

  6.      <comp:item value="4" text="Construction"/>
    7. 

  7.      ...
    8. 

  8.    </comp:listbox>
    9.

We can also setup a listbox within the eventhandler code (typically within the beforeRender (each time the view is rendered) or within the component create event (once the component is created).

i.e. 

  1.    public function beforeRender() {
    2. 

  2.        $listbox = $this->getComponent('categories');
    3. 

  3.        $listbox->addItem(new __Item(1, 'Agriculture, forestry and fishing'));
    4. 

  4.        $listbox->addItem(new __Item(2, 'Arts, sports and recreation'));
    5. 

  5.        $listbox->addItem(new __Item(3, 'Catering and accommodation'));
    6. 

  6.        $listbox->addItem(new __Item(4, 'Construction'));
    7. 

  7.        ...
    8. 

  8.    }
    9.

We can select more than one item within a listbox, which represents the value for that component. For that purpose, we can:

  • Use the __ItemListComponent::setSelectedIndex() method or
  • Set as selected the item previous to add to the listbox (by ussing the __ItemComponent::setSelected() method) or
  • Use the selected property within the item to be selected to
i.e. 
  1.    <comp:listbox name="categories">
    2. 

  2.      <comp:item value="1" text="Agriculture, forestry and fishing" selected="true"/>
    3. 

  3.      <comp:item value="2" text="Arts, sports and recreation" selected="true"/>
    4. 

  4.      <comp:item value="3" text="Catering and accommodation"/>
    5. 

  5.      <comp:item value="4" text="Construction"/>
    6. 

  6.      ...
    7. 

  7.    </comp:listbox>
    8.

A listbox, as a value holder, returns the value of the selected item as a result of call to the getValue.

i.e. 

  1.    public function categories_change(__UIEvent &$event) {
    2. 

  2.       //get the value of the selected item within the combo:
    3. 

  3.       $listbox_value = $event->getComponent()->getValue();
    4. 

  4.  
    5. 

  5.       //...
    6. 

  6.    }
    7.
}

Default events associated to a listbox are the same as associated to a html select element. Most usefull is the change event, raised when the user change the selection.


See the listbox component in action here: http://www.lionframework.org/components/listbox.html

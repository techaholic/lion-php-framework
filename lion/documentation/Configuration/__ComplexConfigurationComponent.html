getComponent


  mixed &getComponent( [string $type = null], [mixed $name = null], [mixed $content = null], [ $attributes = null], [int $index = -1]  )

Tries to find the specified component(s) and returns the objects.

Examples: $propertys =& $obj->getComponent('property'); $property_bar_4 =& $obj->getComponent('property', 'bar', null, 4); $section_foo =& $obj->getComponent('section', 'foo');

This method can only be called on __ComplexConfigurationComponent class. Note that root is a section. This method is not recursive and tries to keep the current structure. For a deeper search, use searchPath()



Parameters:
TypeNameDescription
string $type Type of component: property, section, comment, blank...
mixed $name Component name
mixed $content Find component with this content
$attributes Find component with attribute set to the given value
int $index Index of the component in the returned object list. If it is not set, will try to return the last component with this name.

Return:  reference to component found or false when not found
See:  &searchPath()

getPropertyContent


  mixed getPropertyContent( mixed $args, [int $index = -1]  )

Return a property's content.

This method can use two different search approach, depending on the parameter it is given. If the parameter is an array, it will use the __ComplexConfigurationComponent::searchPath() method. If it is a string, it will use the __ComplexConfigurationComponent::getComponent() method.

Example: 

  1.  // Will return the value found at :
    2. 

  2.  // [Database]
    3. 

  3.  // host=localhost
    4. 

  4.  echo $configuration_component->getPropertyContent(array('Database', 'host')));
    5. 

  5.  
    6. 

  6.  // Will return the value found at :
    7. 

  7.  // date="dec-2004"
    8. 

  8.  echo $configuration_component->getPropertyContent('date');
    9.



Parameters:
TypeNameDescription
mixed $args Search path and attributes or a property name
int $index Index of the component in the returned property list. Eventually used if args is a string.

Return:  Content of property or false if not found.
Access:  public

searchPath


  mixed &searchPath( mixed $args  )

Finds a node using XPATH like format.

The search format is an array: array(component1, component2, component3, ...)

Each component can be defined as the following: component = 'string' : will match the container named 'string' component = array('string', array('name' => 'xyz')) will match the container name 'string' whose attribute name is equal to "xyz" For example : <string name="xyz">



Parameters:
TypeNameDescription
mixed $args Search path and attributes

Return:  Config_Container object, array of Config_Container objects or false on failure.
Access:  public

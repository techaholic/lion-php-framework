Objetive

The objetive of this tutorial is to learn how to apply I18n capabilities to a Yawar application.
The application that we are going to develop just shows a 'Hello world' message.

What we are going to do in this tutorial is:

Executing the Hello World

Now it's time to execute our application, and for that purpose, we need to know how to build the appropriate URL to execute the action we want.


The default mapping rule for URLs in Yawar is <controller>.<action>.html

i.e. if we want to execute the action bar and the controller foo, just use the url foo.bar.html.

For that purpose, just use the following url: http://yourdomain/pathtoyourapplication/helloWorld.html

Yawar will deduce that the action controller that you are requesting is the one with code 'helloWorld'. Because we haven't define any action to execute, Yawar will try to execute one of the following action codes:

  • An action with the same code than the controller, in our example helloWorld.
  • If (and only if) that action doesn't exists, the action with the code 'default', it is, a method named 'defaultAction'

Because we have already declared a method with the name helloWorldAction, this is the method that will be executed in response to the request
